Claude: Review & Retrieval Tool

A four-stage guide to building your first classroom retrieval-practice website with Claude.

Before we begin

The why behind today's workshop

A few minutes here will shape everything that follows.

By the end of today, you'll have made a start on your own review and retrieval tool. A small website you can use at the start of any lesson to bring back what students have learned. You may not finish it today, and that's okay. The aim is to leave with the skills and confidence to keep building.

Before we begin, it's worth being clear on what makes a review and retrieval tool actually work. The principles below are drawn from cognitive science research on how memory and learning happen, and they're baked into the prompts you'll use in Stage 2.

What makes a good review and retrieval tool

01
Low stakes

No grades, no scores, no public ranking. A built-in time limit creates a sense of focus without pressure.

02
Desirable difficulty

Questions are pitched at the right level of difficulty. Challenging enough that students have to think, but accessible enough that they can engage.

03
Interleaving

The tool allows you to mix questions from different topics in the same session, helping students see connections across topics and apply what they know in new situations.

04
Spacing

The tool allows you to revisit topics over weeks and months, not just the unit you're currently teaching, helping build durable, long-term memory.

Stage one

Create your question banks

Do this part outside of Claude. It saves tokens.

1.Create a question bank for Topic A

Prepare a set of 10–15 low-stakes review and retrieval questions for one topic. Ensure that:

  • answers are accurate and verified
  • questions are suitable for quick retrieval practice
  • the mix includes a balance of quick-recall questions and ones that require deeper thinking

2.Create a question bank for Topic B

Prepare a second set of 10–15 questions for another topic, following the same principles.

Stage two

Use Claude to build the tool

Before you prompt Claude, sketch out what you want.

Plan before you prompt

Think about:

  • how you want questions to generate
  • whether questions appear one at a time or all together
  • how answers should be revealed
  • how interleaving should work across topics

Example prompts for Claude

Two starting points depending on what suits your subject. Copy whichever fits, then replace the placeholders at the end with your own question banks from Stage 1.

Prompt A · multi-topic, interleaved
I want to create a review and retrieval website for classroom use. The idea is that when I click on a topic, the website generates random questions from that topic's question bank. The website should always generate exactly 4 questions total. Examples: - If 1 topic is selected → all 4 questions come from that topic - If 2 topics are selected → the 4 questions are mixed across both topics - If multiple topics are selected → the 4 questions should be distributed across those selected topics for interleaving and spaced retrieval practice Because questions are generated randomly, there should not be a "Next Question" button. Answers should remain hidden until I click a "Show All Answers" button located in the top-right corner. The question layout is very important: - Questions should maximise screen real estate - A 2 × 2 card layout is preferred - The cards should scale appropriately for classroom projectors and laptops - Avoid excessive whitespace or tiny centred cards - The design should feel intentional and polished rather than looking like a generic template or dashboard The website should include: - a 5-minute timer - pause and restart functionality - a reset button to clear the session and start again - responsive design for classroom display - clean readability from the back of a classroom Before designing the interface, ask me what visual style/theme I want the website to have. For example, ask about: - colour palette - modern vs playful style - minimalist vs game-like appearance - typography preferences - animations/effects - whether the design should feel academic, futuristic, sporty, retro, etc. The overall design should prioritise: - simplicity - readability - fast classroom use - strong visual presentation - efficient use of screen space Here are the questions and answers for Topic A: (insert) Here are the questions and answers for Topic B: (insert) Before building the website, ask me any clarifying questions you need.
Prompt B · single topic to start
I want to create a review and retrieval website for classroom use. I'll start with ONE topic, but I want the website built so that I can easily add more topics later. For this first version, each question is skills-based rather than recall-based, so questions may be longer (a sentence, a short paragraph, or include a short extract for the student to analyse). The idea is that when the website loads, it randomly selects and displays ONE question from the topic's question bank. The website should always display exactly ONE question at a time, taking up most of the screen. Because the question is generated randomly, there should not be a "Next Question" button. To get a new question, the user clicks the Reset button — this clears the session and randomly draws another question from the bank. The answer should remain hidden until I click a "Show Answer" button located in the top-right corner. The question layout is very important: - The question should maximise screen real estate - A single large card centred on the page is preferred - The card should scale appropriately for classroom projectors and laptops - The question text should be large enough to read from the back of the classroom - If the question includes an extract or quote, present it clearly distinguished from the actual prompt (e.g. in a tinted block or with quotation styling) - Avoid excessive whitespace or tiny centred cards - The design should feel intentional and polished rather than looking like a generic template The website should include: - a 5-minute timer - pause and restart functionality - a reset button to clear the session and draw a new random question - responsive design for classroom display - clean readability from the back of a classroom Important — design for future expansion: - Structure the code so that adding more topics later is straightforward (e.g. a clearly separated topic/question bank section) - When more topics are added in future, I may want to switch to selecting multiple topics and showing 4 questions in a 2×2 grid for interleaving — please keep this in mind when designing the underlying structure Before designing the interface, ask me what visual style/theme I want the website to have. For example, ask about: - colour palette - modern vs playful style - minimalist vs editorial appearance - typography preferences (serif suits English; sans-serif may suit other subjects) - animations/effects - whether the design should feel academic, literary, contemporary, etc. The overall design should prioritise: - simplicity - readability of longer-form questions - fast classroom use - strong visual presentation - efficient use of screen space Here is the topic name and its question bank: Topic name: (insert) Questions and answers: (insert) Before building the website, ask me any clarifying questions you need.
Stage three

Refine and improve

Claude will usually generate an initial mock-up or prototype. The real work happens in the iteration.

From there, continue prompting Claude to:

  • tweak layouts and design
  • improve usability
  • change how questions generate
  • adjust timers and controls
  • refine colours, themes, and styling

Treat the first version as a draft. Be specific about what you want changed. "Make the timer bigger" works, but "make the timer 1.5× larger and move it to the top-centre so it's visible from the back of the room" works better.

Stage four

Save and keep building

Once you're happy with how your tool works, save a copy. Then think about what comes next.

Save your tool

Once you're happy with how the tool functions, download the HTML file onto your desktop. This becomes the version you use in class.

If you want to make changes later (new questions, new topics, a different colour scheme), go back into Claude, ask for the changes, and download the new HTML file. The version on your desktop is a snapshot, not a live document, so each round of edits needs a fresh download.

Quick tip

Keep your question banks in a separate document (Word, OneNote, a notes app). That way, when you want to add new questions to Claude, you don't have to retype them.

Future improvements

Your first version is a foundation. Later, you can:

  1. Expand your question banks
  2. Add varying difficulty levels (e.g. Foundation, Core, Advanced)
  3. Improve visual design (colour palette, themes, animations, fonts)
  4. Add more topics and year levels
  5. Add explanations or worked solutions
  6. Add question types such as:
    • multiple choice
    • error analysis
    • short response
    • problem solving
    • challenge questions
